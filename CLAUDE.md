# DayStep 프로젝트 작업 관리 규칙

## 핵심 원칙

모든 작업은 TASKS.md 파일을 중심으로 진행합니다.

---

## 작업 진행 프로토콜

### 작업 시작 전
1. TASKS.md 읽기
2. 우선순위 및 의존성 파악
3. 할당된 서브 에이전트 확인 (TASKS.md의 "Assigned" 참조)

### 작업 진행 중
- 작업 완료: 체크박스 `[ ]` → `[x]` 변경
- 완료 Task에 추가 정보 기록 (선택사항):
  - 완료 시간
  - 수정 파일 경로
  - 주요 변경 사항

### Phase 순서 준수
- Phase 순차 진행 (N → N+1)
- Dependencies 확인 필수
- 우선순위: 높음 > 중간 > 낮음

---

## 절대 원칙

1. 작업 시작 전 TASKS.md 반드시 확인
2. 할당된 서브 에이전트 사용 (TASKS.md "Assigned" 확인)
3. 작업 진행/완료 시 즉시 TASKS.md 업데이트
4. Phase 순서 및 의존성 준수
5. **할일, 할일 목록 생성/작업 계획 시 task-planner 에이전트 사용 필수**
