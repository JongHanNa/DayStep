  • 필터 기본 숨김 → 검색 아이콘 토글로 축소 (인지 부하 감소)
  • borderLeft 제거 → 감정뱃지 단일 인코딩으로 단순화

  • 핀 노트 → "오늘의 원동력" 배너로 분리 (시각적 계층 생성)

❌ 현재 문제점

  • 감정태그 선택이 4곳에서 반복 (인라인/필터/입력시트/상세시트)
  • "미처리/처리됨" 필터 의미 불명확 → 사용자 혼란
  • ADHD 사용자에게 필터/검색 과도 → 인지 부하 증가
  • 빈 상태에서 감정태그가 필수 아닌데 노출 → 선택 피로

✅ 시안 C 개선점

  • 감정 탭이 필터+분류를 동시 해결 → 별도 필터 불필요
  • 감정 먼저 선택 → 입력 UI가 해당 색상으로 변환 (탭 선택해보세요)
  • 감정별 미니 차트로 통계 시각화 (XP/레벨 게이미피케이션 대체)
  • DisclosureGroup 스타일 확장/축소 → 정보 과부하 방지 (그룹 헤더 클릭)
  • 검색/상태필터 제거 → 감정 탭 하나로 네비게이션 통합

📱 네이티브 컨트롤 교체

  • 감정칩 선택 토글 → UISegmentedControl 스타일 탭 (스크롤 가능)
  • JS 아코디언 → SwiftUI List expandable section
  • {visible && ...} → 네이티브 스프링 트랜지션
  • 감정별 그룹이 자연스러운 분류 역할 → 별도 필터 UI 불필요
