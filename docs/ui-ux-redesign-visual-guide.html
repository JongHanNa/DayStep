1. 변경 요청의 3가지 레이어

요청할 때 어떤 레이어를 바꾸고 싶은지 명시하면 에이전트가 정확히 이해합니다.

UI (Visual Design)

시각적 스타일링 — 색상, 폰트, 간격, 그림자, 라운딩

예: "버튼 색상을 보라색으로", "카드에 글래스모피즘 적용"

UX (Interaction Design)

인터랙션 패턴 — 사용자가 조작하는 방식

예: "+/- 버튼 → 스크롤 휠 피커로 변경"

IA (Information Architecture)

정보 구조 — 콘텐츠 배치, 계층, 네비게이션 흐름

예: "옵션들을 컨텍스트 메뉴로 묶기"

Interactive: 하나의 모달을 3가지 관점으로 보기

4. 변경 규모별 용어

요청의 규모에 따라 적절한 용어를 사용하세요. 각 단계에 호버하면 설명이 표시됩니다.

Tweak / Polish

미세 조정
간격, 색상, 폰트 등 소소한 수정. 구조나 인터랙션은 그대로 유지하면서 시각적 디테일만 다듬는 작업.

Restyling

UI만 변경
같은 구조와 인터랙션에서 비주얼만 교체. 예: 다크 테마 적용, 글래스모피즘 스타일로 전환.

Redesign

UI + UX 변경
시각 디자인과 인터랙션 패턴을 함께 변경. 새로운 조작 방식이나 흐름을 도입.

Overhaul

전면 재설계
UI + UX + IA + 기능 전체 재구성. 정보 구조부터 다시 설계하는 대규모 작업.

Design System
Migration

시스템 전환
전체 앱의 디자인 시스템 교체. 토큰, 컴포넌트 라이브러리, 패턴 전체를 새 시스템으로 이전.
