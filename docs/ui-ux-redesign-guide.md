# UI/UX 리디자인 요청 가이드

> AI 에이전트에게 UI/UX 변경을 요청할 때 사용할 수 있는 전문 용어와 프레임워크

---

## 1. 변경 요청의 5가지 레이어

요청할 때 **어떤 레이어를 바꾸고 싶은지** 명시하면 에이전트가 정확히 이해함.

| 레이어 | 의미 | 예시 |
|--------|------|------|
| **UI (Visual Design)** | 시각적 스타일링 — 색상, 폰트, 간격, 그림자, 라운딩 | "버튼 색상을 보라색으로", "카드에 글래스모피즘 적용" |
| **UX (Interaction Design)** | 개별 요소의 인터랙션 패턴 — 사용자가 조작하는 방식 | "+/- 버튼 → 스크롤 휠 피커로 변경" |
| **IA (Information Architecture)** | 정보 배치, 계층, 네비게이션 구조 | "옵션들을 컨텍스트 메뉴로 묶기", "설정을 탭 구조로 분리" |
| **Flow (Task Flow)** | 목표 달성의 단계별 흐름, 화면 전환 시퀀스 | "3단계 진입 → 원클릭 실행으로", "타이머를 플래너에서 바로 시작" |
| **Behavior (Behavioral Design)** | 동기부여, 넛지, 인지부하 관리, 습관 형성 | "선택지 5→3개로 줄여 결정마비 방지", "첫 세션만 환경 준비 노출" |

**레이어 관계**:
- UI·UX·IA는 **"무엇을, 어떻게, 어디에"** 배치하는가
- Flow는 **"A에서 B로 가는 여정"** — 화면 전환 시퀀스, 단계 수, 진입점/탈출점
- Behavior는 **"왜 이렇게 설계하는가"** — 사용자 심리, 동기부여, 인지 부하의 심리적 근거

**요청 템플릿**:
```
"이 [화면/모달/컴포넌트]의 [UI/UX/IA/Flow/Behavior]를 리디자인해줘"
"[UI만/UX만/Flow+Behavior/전체 5레이어] 변경하고 싶어"
```

## 2. 리디자인 컨셉 키워드

에이전트에게 **방향성**을 전달할 때 쓸 수 있는 키워드들.

### 비주얼 스타일 컨셉
| 키워드 | 설명 |
|--------|------|
| **Glassmorphism** | 반투명 유리 효과, 블러 배경 |
| **Neumorphism** | 부드러운 양각/음각 입체감 |
| **Flat / Minimal** | 그림자 없이 깔끔하고 단순한 스타일 |
| **Material Design** | Google 스타일, 그림자 계층, 리플 효과 |
| **Skeuomorphism** | 실제 사물 질감을 모방한 디자인 |
| **Brutalism** | 굵은 타이포, 강한 대비, 날 것의 느낌 |
| **Organic / Soft UI** | 둥근 모서리, 부드러운 색상, 자연스러운 곡선 |

### 인터랙션 패턴 컨셉
| 키워드 | 설명 |
|--------|------|
| **Direct Manipulation** | 드래그, 스와이프 등 직접 조작 |
| **Progressive Disclosure** | 정보를 단계적으로 노출 (기본 → 상세) |
| **Contextual Actions** | 맥락에 맞는 액션만 노출 (롱프레스 메뉴, 팝오버 등) |
| **Inline Editing** | 별도 화면 없이 제자리에서 편집 |
| **Wizard / Step-by-step** | 단계별 입력 흐름 |
| **Bottom Sheet Cascade** | 바텀시트 위에 바텀시트 (계층적 모달) |
| **Gesture-first** | 제스처 중심 조작 (스와이프, 핀치 등) |

### 인터랙션 다이나믹스 (Interaction Dynamics)

인터랙션 패턴이 **무엇을 하는가**라면, 다이나믹스는 **어떻게 느껴지는가** — 속도, 물리, 연속적 매핑 등 인터랙션의 동작 방식을 정의함.

| 키워드 | 설명 | 예시 |
|--------|------|------|
| **Threshold Trigger** | 조건 충족 시 즉시 발동 (이진) | 엣지 닿으면 바로 페이지 전환 |
| **Continuous Mapping** | 입력량에 비례한 연속적 반응 | 엣지 근접도 → 전환 속도 비례 |
| **Spring Physics** | 탄성 기반 바운스/스냅/오버슈트 | 카드 놓을 때 스프링 효과로 제자리 |
| **Momentum / Inertia** | 관성 기반 감속 | 빠르게 스크롤 후 점차 멈춤 |
| **Haptic Feedback** | 물리 반응을 촉각으로 전달 | 드래그 중 엣지 진입 시 진동 |
| **Dead Zone** | 의도치 않은 입력 방지 영역 | 엣지에서 일정 거리 안에선 무반응 |

### 정보 구조 컨셉
| 키워드 | 설명 |
|--------|------|
| **Hub & Spoke** | 메인 허브에서 세부 화면으로 분기 |
| **Flat Navigation** | 모든 항목이 같은 레벨 (탭 바 등) |
| **Hierarchical** | 트리 구조로 깊어지는 네비게이션 |
| **Card-based** | 카드 단위로 정보 그룹핑 |
| **Dashboard** | 핵심 지표를 한 화면에 요약 |

### 태스크 플로우 컨셉 (Task Flow)

IA가 "어디에 무엇이 있는가"라면, Flow는 **"A에서 B로 가는 여정"** — 화면 전환 시퀀스, 단계 수, 진입점/탈출점, 분기 조건을 다룸.

| 키워드 | 설명 | 예시 |
|--------|------|------|
| **Entry Point Reduction** | 목표 달성까지의 진입 단계 최소화 | 추천→환경준비→타이머 3단계 → 원클릭 실행 |
| **Shortcut Path** | 빈번한 작업에 대한 지름길 제공 | 플래너에서 바로 타이머 시작 버튼 |
| **Context Preservation** | 화면 전환 시 이전 맥락 유지 | 타이머 중 플래너 미니바 표시 |
| **Smart Default** | 반복 설정을 자동으로 채워주기 | 환경 준비를 첫 세션만 노출, 이후 자동 스킵 |
| **Progressive Onboarding** | 처음엔 가이드, 익숙해지면 간소화 | 첫 사용 시 전체 플로우, 이후 축약 |

### 행동 설계 컨셉 (Behavioral Design)

사용자 심리, 동기부여 패턴, 인지 부하를 다룸. ADHD 앱에서 특히 중요 — **"왜 이렇게 설계하는가"**의 심리적 근거.

| 키워드 | 설명 | 예시 |
|--------|------|------|
| **Decision Reduction** | 선택지를 줄여 결정 마비 방지 | 5개 액션 버튼 → 3개로 축소 |
| **Implementation Intention** | "언제-어디서-무엇을" 구체적 행동 계획 | 할일에 시간+장소 결합 |
| **Ritual Design** | 의식(ritual) 단계로 모드 전환 유도 | 환경 준비 체크리스트로 집중 모드 진입 |
| **Positive Friction** | 의도적 마찰로 의미 있는 행동 유도 | 완료 전 간단한 회고 질문 |
| **Nudge** | 부드러운 유도로 바람직한 행동 촉진 | "지금 가장 작은 것 하나" 추천 |
| **Commitment Device** | 사전 약속으로 실행력 강화 | 타이머 시작 = 집중 약속 |

## 3. 요청 예시 모음

### 가벼운 요청 (UI만)
```
"이 모달을 Glassmorphism 스타일로 바꿔줘"
"버튼들을 Pill shape + 보라색 액센트로 변경해줘"
"전체적으로 Soft UI 느낌으로 리스타일링해줘"
```

### 중간 요청 (UI + UX)
```
"시간 선택을 +/- 버튼 대신 스크롤 휠 피커로 바꾸고,
 프리셋 칩 버튼도 추가해줘"
"이 설정 화면을 Progressive Disclosure 패턴으로 리디자인해줘
 — 기본 옵션만 보이고, 고급 옵션은 '더보기'로 숨겨줘"
```

### 큰 요청 (UI + UX + IA)
```
"이 할일 편집 모달의 UI/UX/IA를 전면 리디자인해줘.
 - 비주얼: Glassmorphism + 보라색 액센트
 - 인터랙션: Contextual Actions + Bottom Sheet Cascade
 - 정보구조: Progressive Disclosure로 단순화"
```

### 인터랙션 다이나믹스 요청
```
"DnD 엣지 스크롤을 Continuous Mapping으로 — 엣지에 가까울수록 빠르게"
"리스트 스크롤에 Momentum 효과 추가 — iOS 네이티브 느낌으로"
"드래그 시작에 Dead Zone 20px 적용 — 탭과 구분되게"
```

### 플로우 변경 요청 (Flow)
```
"집중 실행 진입을 3단계에서 1클릭으로 줄여줘 — Entry Point Reduction"
"타이머 실행 중에도 플래너 맥락이 보이게 해줘 — Context Preservation"
"환경 준비 단계를 첫 세션만 노출하고 이후엔 스킵해줘 — Smart Default"
"플래너에서 할일 옆에 바로 타이머 시작 버튼 추가해줘 — Shortcut Path"
```

### 행동 설계 요청 (Behavior)
```
"추천 화면 선택지를 5개에서 3개로 줄여줘 — Decision Reduction"
"집중 시작 전 환경 준비를 의식(ritual)처럼 디자인해줘 — Ritual Design"
"타이머 완료 후 부드럽게 다음 할일로 유도해줘 — Nudge"
"ADHD 사용자의 시작 저항을 줄이는 원클릭 실행 패턴 적용해줘"
```

### 컨셉 제안 요청
```
"이 화면의 UI/UX 리디자인 컨셉을 3가지 제안해줘"
"이 모달에 적용 가능한 인터랙션 패턴 옵션들을 보여줘"
"현재 화면의 UX 문제점을 분석하고 개선안을 제시해줘"
"이 화면의 Flow를 분석하고 단계 축소 방안을 제안해줘"
"ADHD 사용자 관점에서 Behavior 개선안을 제시해줘"
```

## 4. 변경 규모별 용어

| 용어 | 규모 | 설명 |
|------|------|------|
| **Tweak / Polish** | 미세 조정 | 간격, 색상, 폰트 등 소소한 수정 |
| **Restyling** | UI만 변경 | 같은 구조에서 비주얼만 교체 |
| **Redesign** | UI + UX 변경 | 시각 + 인터랙션 패턴 변경 |
| **Flow Optimization** | Flow 변경 | 단계 축소, 진입점 추가, 화면 전환 개선 |
| **Behavioral Redesign** | Behavior 변경 | 동기부여·넛지·인지부하 설계 개선 |
| **Overhaul** | 전면 재설계 | UI + UX + IA + Flow + Behavior 전체 재구성 |
| **Design System Migration** | 시스템 전환 | 전체 앱의 디자인 시스템 교체 |
