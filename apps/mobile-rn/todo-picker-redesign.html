할일 연결 피커 — 리디자인 시안

원동력 새기기 화면에서 할일을 연결/해제하는 피커 모달

현재 화면 문제 분석

시각적 스타일링

할일 제목과 체크 원이 수직 정렬 불일치. 항목 간 간격이 불균일하고 텍스트가 체크 원보다 위에 배치됨. 전체적으로 리스트 밀도가 낮아 스크롤 양 증가.

정보 구조

일반 할일과 반복 할일 구분 없이 단일 리스트로 나열. 이미 연결된 할일이 어떤 것인지 시각적 구분 없음. 항목이 많을 때 원하는 할일을 찾기 어려움.

인터랙션 패턴

RN Modal (presentationStyle="pageSheet")을 사용하나 검색/필터 없이 전체 목록만 나열. ScrollView 내 스크롤은 네이티브 리스트 대비 물리감이 부족하고 rubber-band가 어색.

태스크 플로우

연결 상태 변경 후 피드백 없음 — 체크만 토글되고 "n개 연결됨" 같은 요약이 없음. 완료 후 결과 확인을 위해 모달을 닫아야 하는 구조.

행동설계

"연결" 행위의 의미가 불명확 — 왜 이 할일을 원동력에 연결하는지 맥락 제공 없음. 빈 체크 원만으로는 연결/미연결 상태의 시각적 차이가 약함.

A. 네이티브 그룹 리스트
iOS UISearchController + UITableView grouped 스타일.

네이티브 컨트롤 교체 요약

RN Modal → SwiftUI .sheet

.presentationDetents([.medium, .large])로 네이티브 시트 제스처 (드래그 dismiss, 러버밴드, 속도 감응) 확보. 현재 RN Modal의 pageSheet는 제스처 미지원.

RN ScrollView → SwiftUI List

List(.insetGrouped) 또는 List(.plain)으로 네이티브 스크롤 물리감, 셀 재활용, 섹션 헤더 sticky 효과 확보. 대량 항목에서 성능 이점.

JS 검색 → SwiftUI .searchable

.searchable modifier로 네이티브 검색바 UI + 키보드 자동 dismiss + 취소 버튼 자동 생성. JS TextInput 검색바 대비 플랫폼 일관성 확보.

JS 체크 토글 → SwiftUI Toggle / .swipeActions

연결 해제는 .swipeActions(edge: .trailing)로 스와이프 삭제 패턴 활용. 연결 추가는 탭 시 네이티브 haptic + 스프링 애니메이션 .withAnimation(.spring).

JS 필터 버튼 → SwiftUI Picker(.segmented)

시안 B의 타입 필터를 네이티브 세그먼트 컨트롤로 구현. 터치 피드백, 슬라이드 전환이 플랫폼 네이티브 수준.
