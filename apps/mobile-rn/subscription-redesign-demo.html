1 시각적 스타일링 (Visual Styling)

Critical

플랫폼 구분 부재

구독이 웹(Paddle), App Store, Play Store 중 어디서 결제되었는지 시각적 표시가 전혀 없음. subscriptionInfo.platform 필드가 존재하지만 UI에 미반영.

→ 사용자가 자신의 구독 출처를 인지하지 못해 관리 불가
Warning

플랜 정보 미표시

월간(₩5,500)/연간(₩44,000) 플랜 구분과 가격 정보 미표시. productId 필드(pro_monthly/pro_yearly) 미활용.

→ "DayStep Pro"만 표시되어 어떤 플랜인지 알 수 없음
Warning

상태 카드 정보 빈약

Crown 아이콘 + 제목 + 날짜만 표시. 웹은 가격·플랫폼·플랜·자동갱신·결제수단 등 풍부한 정보 제공.

→ 웹 대비 정보 밀도가 현저히 낮음
Info

Pro/Free 비교표 과잉

이미 Pro인 사용자에게도 비교표가 화면의 대부분을 차지. 하드코딩된 문자열("60개", "무제한")로 현재 사용량 미반영.

→ 이미 Pro 구독 중인 사용자에게 불필요한 정보 강조

2 인터랙션 패턴 (Interaction Pattern)

Critical · Core Bug

"구독 관리" 버튼 오작동

openSubscriptionManagement()가 Platform.OS만 참조. iOS이면 무조건 App Store로, Android이면 Play Store로 이동. subscriptionInfo.platform 완전 무시.

→ 웹 구독자가 "구독 관리" 누르면 App Store(구독 없음)로 이동 → Dead End
Critical

업그레이드 CTA도 동일 문제

Free 사용자의 "Pro로 업그레이드" 버튼도 무조건 App Store/Play Store로만 이동. Paddle(웹) 결제 경로 없음.

→ 인앱 구매가 구현되지 않은 상태에서 Store로 보내면 구매 불가
Warning

구독 복원 기능 없음

웹의 handleRestore()에 대응하는 기능 미구현. 기기 변경 시 구독 복원 불가.

→ 기기 교체 후 Pro 상태 인식 실패 가능성
Warning

구독 취소/변경 불가

웹의 Paddle API(/api/paddle/manage) 연동 없음. 결제수단 변경, 플랜 업/다운그레이드 불가.

→ 모든 관리 행위가 외부 링크에 의존

3 정보 구조 (Information Architecture)

Critical

결제 플랫폼 미표시

DB에 platform 필드('web'/'ios'/'android') 존재하지만 UI에 표시 안 됨.

→ 웹: "Paddle (웹)" 명시 vs RN: 아무 표시 없음
Warning

도움말/환불 섹션 없음

환불 정책(7일 이내), 이메일 문의 경로, 이용약관 링크 없음.

→ 웹: 섹션 3에서 환불 정책 + 이메일 안내 제공
Info

크로스 플랫폼 안내 부재

"다른 플랫폼에서 구독하셨나요?" 안내 없음. 웹에서는 비구독자에게 App Store/Play Store 안내.

→ 플랫폼 간 구독 인식 경로 부재
Info

단일 계층 구조

모든 정보가 평면적으로 나열. 웹은 3섹션 아코디언으로 체계적 분류.

→ 필요한 정보를 찾기 어려움

4 태스크 플로우 (Task Flow)

🔴 현재: 웹 구독자의 모바일 여정

Settings → 구독 관리 → "DayStep Pro" 확인 → "구독 관리" 클릭 → App Store 열림 → 구독 없음 😕 → 돌아옴... 어디서 관리?

🟢 이상적: 플랫폼 인식 여정

Settings → 구독 관리 → "Pro · 연간 · 웹 결제" 확인 → "웹에서 결제됨" 안내 → 웹 관리 링크 or 이메일 문의
Task Failure

구독 상태 확인

부분 성공 — Pro인 건 알지만 플랜(월간/연간), 가격, 결제 플랫폼을 알 수 없음.

Task Failure

구독 관리

완전 실패 — 잘못된 목적지(App Store)로 이동. 관리 불가.

Task Failure

플랜 변경

불가 — 월간↔연간 변경 경로 자체가 없음.

Task Failure

도움 요청

불가 — 이메일 연락처, 고객센터 링크 없음.

5 행동 설계 (Behavioral Design)

Critical

거짓 행동유도성 (False Affordance)

"구독 관리" 버튼이 구독을 관리할 것으로 기대시키지만, 전혀 관계없는 App Store로 이동.

→ 시스템에 대한 신뢰 상실
Warning

학습된 무력감

반복적 실패 → "모바일에서는 관리가 안 되는구나" 체념.

→ 사용자가 구독 관리를 포기
Warning

인지 부하 가중

자신의 구독이 어디서 결제됐는지 사용자가 직접 기억해야 함. 시스템이 알려주지 않음.

→ 불필요한 인지 부하 → 오류 유발
Warning

복구 경로 부재

잘못된 곳(App Store)으로 갔을 때 되돌아올 안내 없음.

→ 이탈 → 고객 문의 증가

🔧 근본 원인 코드

// SubscriptionView.tsx 26-32행
function openSubscriptionManagement() {
  const url =
    Platform.OS === 'ios'
      ? 'https://apps.apple.com/account/subscriptions'
      : 'https://play.google.com/store/account/subscriptions';
  Linking.openURL(url);
}

// ❌ subscriptionInfo.platform 완전 무시!
// ❌ 'web' 플랫폼 처리 없음!
// ❌ Paddle 관리 URL 연동 없음!

상태 전환

시안 A 컨셉

"플랫폼 인식 카드" — 구독 플랫폼을 최우선으로 인식시키고, 플랫폼별 적절한 관리 경로를 제공합니다.

핵심 개선점

  • ✓ 플랫폼 배지 ("웹 결제" / "App Store")
  • ✓ 플랜 유형 + 가격 표시
  • ✓ 결제일·자동갱신 정보 그리드
  • ✓ 플랫폼별 관리 버튼 분기
  • ✓ 알림 배너 (결제 출처 안내)
  • ✓ 비교표 접힌 상태로 이동

상태 전환

시안 B 컨셉

"스마트 대시보드" — 웹 버전과 동일한 3섹션 아코디언 구조를 모바일에 최적화하여 제공합니다.

핵심 개선점

  • ✓ 3섹션 아코디언 (정보·관리·도움말)
  • ✓ 플랫폼별 액션 버튼 분기
  • ✓ 환불 정책 + 이메일 문의 섹션
  • ✓ 크로스 플랫폼 안내 하단
  • ✓ 웹과 일관된 정보 구조

상태 전환

시안 C 컨셉

"컨텍스트 가이드" — 상태 기반 적응형 UI. 사용자의 구독 플랫폼에 따라 화면 전체가 달라집니다.

핵심 개선점

  • ✓ 히어로 상태 카드 (가격 강조)
  • ✓ 플랫폼별 "다음 단계" 가이드
  • ✓ 적응형 UI (상태마다 다른 화면)
  • ✓ FAQ 아코디언
  • ✓ 만료 임박 시 경고 색상 변화
  • ✓ Free → 업그레이드 히어로 CTA
