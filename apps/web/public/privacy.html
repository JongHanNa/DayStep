일상투두(이하 "서비스")은 정보주체의 자유와 권리 보호를 위해 「개인정보 보호법」 및 관계 법령이 정한 바를 준수하여, 적법하게 개인정보를 처리하고 안전하게 관리하고 있습니다. 이에 「개인정보 보호법」 제30조에 따라 정보주체에게 개인정보 처리에 관한 절차 및 기준을 안내하고, 이와 관련한 고충을 신속하고 원활하게 처리할 수 있도록 하기 위하여 다음과 같이 개인정보 처리방침을 수립·공개합니다.

서비스는 회원가입, 상담, 서비스 제공 등을 위해 아래와 같은 개인정보를 수집하고 있습니다.

  • 서비스 이용 과정에서 자동 수집 (쿠키, 로그 분석 등)
  • 모바일 애플리케이션 권한 승인을 통한 수집 (연락처, 알림 등)

수탁업체 위탁 업무 내용 보유 및 이용 기간
Supabase Inc. 데이터베이스 호스팅, 사용자 인증, 파일 스토리지 회원 탈퇴 시 또는 위탁 계약 종료 시까지

6. 정보주체의 권리·의무 및 행사 방법

정보주체는 언제든지 다음 각 호의 개인정보 보호 관련 권리를 행사할 수 있습니다.

가. 정보주체의 권리

  • 개인정보 열람 요구
  • 개인정보 정정·삭제 요구
  • 개인정보 처리 정지 요구
  • 개인정보 제공 동의 철회 (회원 탈퇴)

나. 권리 행사 방법

  • 앱 내 설정: 모바일 앱 → 설정 → 계정 관리 → 회원 탈퇴
  • 웹 설정: 웹사이트 → 설정 → 계정 관리 → 회원 탈퇴
  • 이메일 요청: [개인정보 보호책임자 이메일 주소]로 요청

다. 회원 탈퇴 절차

  1. 앱 또는 웹에서 "설정" → "계정 관리" 선택
  2. "회원 탈퇴" 버튼 클릭
  3. 탈퇴 사유 선택 (선택사항)
  4. "탈퇴하기" 확인
  5. 모든 개인정보 즉시 삭제 (복구 불가)

주의사항: 회원 탈퇴 시 작성하신 모든 데이터 (할일, 노트, 프로젝트 등)가 영구적으로 삭제되며 복구할 수 없습니다.

7. 개인정보의 파기 절차 및 방법

서비스는 개인정보 보유기간의 경과, 처리목적 달성 등 개인정보가 불필요하게 되었을 때에는 지체없이 해당 개인정보를 파기합니다.

가. 파기 절차

  • 정보주체로부터 동의받은 개인정보 보유기간이 경과하거나 처리목적이 달성되었음에도 불구하고 다른 법령에 따라 개인정보를 계속 보존하여야 하는 경우에는, 해당 개인정보를 별도의 데이터베이스(DB)로 옮기거나 보관장소를 달리하여 보존합니다.
  • 파기 사유가 발생한 개인정보를 선정하고, 서비스의 개인정보 보호책임자의 승인을 받아 개인정보를 파기합니다.

나. 파기 방법

  • 전자적 파일: 기록을 재생할 수 없는 기술적 방법을 사용하여 삭제 (Supabase RLS를 통한 영구 삭제)
  • 종이 문서: 분쇄기로 분쇄하거나 소각

다. 파기 시점

  • 회원 탈퇴 요청 즉시
  • 서비스 이용 계약 해지 또는 회원 자격 상실 시
  • 개인정보 유효기간 경과 시

8. 개인정보 보호를 위한 기술적·관리적 조치

서비스는 개인정보의 안전성 확보를 위해 다음과 같은 조치를 취하고 있습니다.

가. 기술적 조치

  • 암호화 통신: HTTPS 프로토콜을 통한 모든 데이터 전송 암호화
  • 데이터베이스 보안: Supabase RLS (Row Level Security) 적용으로 사용자별 데이터 격리
  • 인증 토큰 관리: JWT (JSON Web Token) 기반 인증, 자동 갱신 (5분 전)
  • 로컬 저장소 암호화: Capacitor Preferences를 통한 안전한 로컬 데이터 저장
  • 비밀번호 암호화: Supabase Auth의 bcrypt 해시 알고리즘 사용
  • 접근 제어: IP 기반 접근 제어 및 비정상 접근 모니터링

나. 관리적 조치

  • 개인정보 취급 최소화: 개인정보 처리 직원의 지정 및 최소화
  • 정기적 자체 감사: 개인정보 처리 시스템 점검 및 감사 실시
  • 내부 관리계획 수립: 개인정보 보호 내부 관리계획 수립 및 시행
  • 접근 권한 관리: 개인정보 처리 시스템 접근 권한 부여·변경·말소 관리
  • 접속 기록 보관: 개인정보 처리 시스템 접속 기록 보관 및 위조·변조 방지

다. 물리적 조치

  • 클라우드 보안: Supabase의 AWS 기반 데이터 센터 (ISO 27001, SOC 2 인증)
  • 백업 관리: 정기적 데이터 백업 및 복구 절차 수립
  • 재해 대비: 다중 리전 백업을 통한 재해 대비

9. 쿠키 및 로컬 저장소 사용

서비스는 이용자에게 맞춤형 서비스를 제공하기 위해 쿠키(Cookie) 및 로컬 저장소를 사용합니다.

가. 쿠키란?

쿠키는 웹사이트가 이용자의 브라우저에 저장하는 작은 텍스트 파일로, 웹사이트 방문 시 이용자의 설정 정보를 저장하고 불러오는 데 사용됩니다.

나. 서비스의 쿠키 사용 목적

  • 로그인 세션 유지 (Supabase Auth 쿠키)
  • 사용자 인증 상태 확인
  • 서비스 이용 통계 분석

다. 쿠키 설정

쿠키명 목적 유효기간
supabase-auth-token 사용자 인증 (Access Token) 7일
supabase-auth-refresh-token 토큰 자동 갱신 (Refresh Token) 7일

라. 로컬 저장소 사용

모바일 앱에서는 Capacitor Preferences를 사용하여 다음 정보를 로컬에 저장합니다:

  • 인증 세션 정보 (암호화 저장)
  • 앱 설정 정보
  • 오프라인 데이터 캐시

마. 쿠키 거부 방법

이용자는 쿠키 설치에 대한 선택권을 가지고 있습니다. 따라서 웹 브라우저에서 옵션을 설정함으로써 모든 쿠키를 허용하거나, 쿠키가 저장될 때마다 확인을 거치거나, 모든 쿠키의 저장을 거부할 수 있습니다.

  • Chrome: 설정 → 개인정보 및 보안 → 쿠키 및 기타 사이트 데이터
  • Safari: 환경설정 → 개인정보 → 쿠키 및 웹사이트 데이터

주의: 쿠키 저장을 거부할 경우 로그인이 필요한 일부 서비스 이용에 어려움이 있을 수 있습니다.

10. 모바일 애플리케이션 권한
