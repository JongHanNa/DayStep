원동력 시트 UI/UX 리디자인 — 3안 비교 · 네이티브 컨트롤 교체 매핑

현재 MotivationInputBottomSheet (신규 작성)와 MotivationDetailBottomSheet (편집/상세)는 JS TextInput·Pressable·ScrollView 조합으로 구현되어 키보드 회피·스크롤·제스처·시각 계층에서 반복적인 문제를 낳아 왔습니다. 이 문서는 문제를 5개 축으로 진단하고, 할일 연결 시트(NativeTodoPicker)의 네이티브 품질을 기준으로 iOS/Android 네이티브 컨트롤로 교체한 3가지 리디자인 안을 제안합니다.

1현재 문제 진단

시각 스타일링

  • 감정 태그 알약 4개가 전체 공간의 20% 차지, 정보 가치 낮음
  • 제목/본문 시각 위계 거의 없음
  • 풀폭 회색 비활성 저장 버튼 — 대시보드 느낌
  • iOS 26 Liquid Glass 완전 미반영
  • 편집 모드 상단 아이콘 3개 고립 배치, 위계 부재

인터랙션 패턴

  • JS AnimatedPressable 칩 — SwiftUI Picker 네이티브 대체 가능
  • RN TextInput — 네이티브 선택/드래그/Dynamic Type 상실
  • @gorhom/bottom-sheet + BottomSheetScrollView 조합의 스크롤·키보드 이슈 반복
  • KeyboardAvoidingView pageSheet 내부 좌표 계산 이슈

정보 구조

  • 신규 작성 시트와 편집 시트 레이아웃 비일관
  • "연결된 할일" 회색 박스가 카드 위계를 깨뜨림
  • 날짜 헤더가 공간만 차지하고 행동을 유도하지 않음
  • 감정 태그 의미론이 불명확 (이미 제거됨)

태스크 플로우

  • 제목·본문 분리 입력 — 빠른 기록 상황에서 인지 부하
  • 저장 버튼 위치 변경 반복 (하단→상단) 이 플로우 불확정성 방증
  • 키보드에 가려진 하단 접근이 무한 퍼즐화

행동 설계

  • "원동력을 새긴다"는 컨셉과 UI가 단절됨 (단순 CRUD 느낌)
  • 작성 완료 후 피드백/보상 없음
  • 반복 작성 습관 유도 장치 부재
  • 프롬프트/템플릿 같은 scaffolding 없음

2네이티브 컨트롤 교체 원칙 (공통)

모든 3가지 시안은 아래 공통 원칙 위에 세워집니다. JS 레이어의 역할은 네이티브 컴포넌트를 mount하고 이벤트를 수신하는 것으로 축소됩니다.

// NativeMotivationEditor.swift — `src/components/native/NativeMotivationEditor`
import SwiftUI

struct NativeMotivationEditor: View {
  @State var text: String = ""
  @State var isPinned: Bool = false
  let onSave: (String, Bool) -> Void
  let onCancel: () -> Void

  var body: some View {
    NavigationStack {
      TextEditor(text: $text)
        .scrollContentBackground(.hidden)
        .padding(.horizontal, 16)
        .font(.system(size: 16))
        // iOS 26+ Liquid Glass auto-applied to nav bar
        .toolbar {
          ToolbarItem(placement: .cancellationAction) {
            Button("취소", action: onCancel)
          }
          ToolbarItem(placement: .principal) {
            Text("원동력").fontWeight(.semibold)
          }
          ToolbarItem(placement: .confirmationAction) {
            Button("저장") { onSave(text, isPinned) }
              .buttonStyle(.borderedProminent)
              .tint(Color(hex: "#D97706"))
          }
          ToolbarItemGroup(placement: .bottomBar) {
            Button { /* open NativeTodoPicker */ } label: {
              Label("연결", systemImage: "link.badge.plus")
            }
            Spacer()
            Toggle(isOn: $isPinned) {
              Image(systemName: isPinned ? "pin.fill" : "pin")
            }.toggleStyle(.button)
          }
        }
    }
    .presentationDetents([.large])
    .presentationDragIndicator(.visible)
  }
}

// NativeMotivationEditor.kt — android/app/src/main/java/.../native
@OptIn(ExperimentalMaterial3Api::class)
@Composable
fun NativeMotivationEditor(
  onSave: (text: String, pinned: Boolean) -> Unit,
  onCancel: () -> Unit,
) {
  var text by rememberSaveable { mutableStateOf("") }
  var pinned by rememberSaveable { mutableStateOf(false) }
  val sheetState = rememberModalBottomSheetState(skipPartiallyExpanded = true)

  ModalBottomSheet(
    onDismissRequest = onCancel,
    sheetState = sheetState,
    dragHandle = { BottomSheetDefaults.DragHandle() },
  ) {
    Scaffold(
      topBar = {
        TopAppBar(
          title = { Text("원동력") },
          navigationIcon = {
            TextButton(onClick = onCancel) { Text("취소") }
          },
          actions = {
            FilledTonalButton(onClick = { onSave(text, pinned) }) {
              Text("저장")
            }
          }
        )
      },
      bottomBar = {
        BottomAppBar {
          IconButton(onClick = { /* todo picker */ }) {
            Icon(Icons.Outlined.Link, null)
          }
          Spacer(Modifier.weight(1f))
          IconToggleButton(pinned, onCheckedChange = { pinned = it }) {
            Icon(if (pinned) Icons.Filled.PushPin else Icons.Outlined.PushPin, null)
          }
        }
      }
    ) { padding ->
      BasicTextField(
        value = text, onValueChange = { text = it },
        modifier = Modifier.fillMaxSize().padding(padding).imePadding().padding(16.dp),
        textStyle = LocalTextStyle.current.copy(fontSize = 16.sp),
      )
    }
  }
}

// NativeMotivationJournal.swift
struct NativeMotivationJournal: View {
  @State var title = ""
  @State var body = ""
  @State var isPinned = false
  let date: Date
  let linkedTodos: [LinkedTodo]
  let onSave: () -> Void
  let onUnlink: (String) -> Void

  var body: some View {
    NavigationStack {
      Form {
        Section {
          VStack(alignment: .leading, spacing: 4) {
            Text(date, format: .dateTime.year().month().day())
              .font(.system(size: 28, weight: .heavy))
            Text(date, format: .dateTime.weekday(.wide).hour().minute())
              .font(.footnote).foregroundStyle(.secondary)
          }
          Text("🌱 오늘 당신을 움직인 것은 무엇인가요?")
            .font(.subheadline).foregroundStyle(.tint)
          TextField("제목", text: $title)
            .font(.title3.weight(.semibold))
          TextEditor(text: $body)
            .frame(minHeight: 140)
            .overlay(alignment: .topLeading) {
              if body.isEmpty {
                Text("떠오르는 것을 적어보세요…")
                  .foregroundStyle(.tertiary).padding(.top, 8)
              }
            }
        }
        Section("연결된 할일") {
          ForEach(linkedTodos) { todo in
            Label(todo.title, systemImage: "arrow.triangle.2.circlepath")
              .swipeActions(edge: .trailing) {
                Button(role: .destructive) {
                  onUnlink(todo.id)
                } label: { Label("해제", systemImage: "link.badge.minus") }
              }
          }
          Button { /* present NativeTodoPicker */ } label: {
            Label("할일 연결", systemImage: "link.badge.plus")
          }
        }
      }
      .toolbar {
        ToolbarItem(placement: .cancellationAction) { Button("닫기", action: {}) }
        ToolbarItem(placement: .primaryAction) {
          Menu {
            Toggle("고정", isOn: $isPinned)
            Button(role: .destructive) {} label: { Label("삭제", systemImage: "trash") }
          } label: { Image(systemName: "ellipsis.circle") }
        }
        ToolbarItem(placement: .confirmationAction) {
          Button("저장", action: onSave)
            .buttonStyle(.borderedProminent)
            .disabled(title.isEmpty && body.isEmpty)
        }
      }
    }
  }
}

        state = rememberSwipeToDismissBoxState(),

        content = { ListItem(headlineContent = { Text(todo.title) }) }
      )
    }
  }
}

// NativeMotivationQuickCapture.swift
struct NativeMotivationQuickCapture: View {
  @State var text = ""
  @State var isPinned = false
  @State var detent: PresentationDetent = .medium
  @FocusState var focused: Bool

  var body: some View {
    NavigationStack {
      VStack(spacing: 0) {
        TextEditor(text: $text)
          .focused($focused)
          .padding(.horizontal, 16)
          .font(.system(size: 16))
        if detent == .large {
          Divider()
          DisclosureGroup("연결된 할일") {
            // forEach linked todos
          }.padding()
        }
      }
      .toolbar {
        // ToolbarItemGroup(.keyboard) → 키보드 바로 위 고정
        ToolbarItemGroup(placement: .keyboard) {
          Button { /* todo picker */ } label: {
            Image(systemName: "link.badge.plus")
          }
          Toggle("고정", isOn: $isPinned)
            .toggleStyle(.button).labelStyle(.iconOnly)
          Spacer()
          Button("저장") { /* save */ }
            .buttonStyle(.borderedProminent)
            .tint(Color(hex: "#D97706"))
            .disabled(text.isEmpty)
        }
      }
    }
    .presentationDetents([.medium, .large], selection: $detent)
    .presentationDragIndicator(.visible)
    .presentationBackgroundInteraction(.enabled(upThrough: .medium))
    .onAppear { focused = true }
  }
}

@OptIn(ExperimentalMaterial3Api::class)
@Composable
fun NativeMotivationQuickCapture(onDismiss: () -> Unit) {
  var text by rememberSaveable { mutableStateOf("") }
  var expanded by remember { mutableStateOf(false) }
  val sheet = rememberModalBottomSheetState(skipPartiallyExpanded = false)

  ModalBottomSheet(
    onDismissRequest = onDismiss,
    sheetState = sheet,
    dragHandle = { BottomSheetDefaults.DragHandle() },
  ) {
    Column(modifier = Modifier.imePadding()) {
      BasicTextField(
        value = text, onValueChange = { text = it },
        modifier = Modifier.fillMaxWidth().padding(16.dp).heightIn(min = 120.dp),
      )
      AnimatedVisibility(visible = expanded) {
        LinkedTodosList()
      }
      Row(
        modifier = Modifier.fillMaxWidth().padding(12.dp),
        horizontalArrangement = Arrangement.SpaceBetween,
      ) {
        Row { /* link / pin icons */ }
        FilledTonalButton(onClick = { /* save */ }, enabled = text.isNotBlank()) {
          Text("저장")
        }
      }
    }
  }
  LaunchedEffect(sheet.currentValue) {
    expanded = sheet.currentValue == SheetValue.Expanded

상황별 선택 가이드

세 안 모두 네이티브 품질을 보장하지만, 제품 목표와 구현 캐파에 따라 선택이 달라집니다. DayStep이 ADHD 사용자의 "원동력 새기기"를 돕는 제품이라는 점에서 다음을 제안합니다.

단기(2주 내) · 최소 리스크

시안 A를 먼저 적용. 네이티브 시트+TextEditor만으로 현재 겪고 있는 스크롤·키보드·탭바 이슈를 전부 해소하고 구현 분량이 가장 적습니다. 기존 DB 스키마와도 완전히 호환됩니다(첫 줄 = title 파싱).

중기 · 브랜드 차별화

시안 B로 업그레이드. 프롬프트 기반 성찰 유도는 "원동력을 새기는" 제품 내러티브와 직접 연결되며, Apple Journal이 검증한 패턴이므로 사용자 학습 비용이 낮습니다. 반복 작성 동기 부여에서 A 대비 명확한 우위.

장기 · 전문가 사용자

시안 C의 키보드 accessory + detent 전환 패턴은 파워유저가 하루 여러 번 캡처하는 시나리오에 최적. 네이티브 제스처를 전면에 드러내 "앱 품질"에 대한 인상을 만듭니다. 최소 iOS 16+ 요구 · Android 대응 추가 설계 필요.
