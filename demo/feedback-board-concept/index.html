한눈에 비교

같은 기능을 세 가지 멘탈 모델로 해석했을 때의 핵심 차이
A · 개인 저널 B · 상태 보드 C · DM 스레드
멘탈 모델 내가 보낸 편지 서류함 내 제보의 칸반 트래커 개발팀과의 1:1 메신저
주 동기 기록 / 주인의식 진척감 답장 / 관계성
타인 게시물 처리 집계 한 줄만 ("48건 제보 중") 섹션별 dimmed row ("비공개 · 14건") 완전 비가시 (모델 내장)
Primary CTA 우하단 FAB · primaryColor 우상단 + · toolbar 우상단 ✎ · Messages 관습
정보 계층 수직 timeline (시간순) 수평 status grouping 스레드 단위 (최근 활동순)
핵심 네이티브 컨트롤 .navigationTitle(.large), List(.plain), .glassEffect() FAB, .sheet(.large) Picker(.segmented), LazyVStack sticky, DisclosureGroup, matchedGeometryEffect .searchable, .badge(), .symbolEffect, List + .swipeActions, Messages chat bubble
신규 브리지 필요 없음 (기존 LiquidGlass + 시트) SegmentedControl 래퍼 .searchable 래퍼 + chat UI
적합한 사용자 기록 자체에 의미를 두는 사용자 제품 운영 신뢰도를 중시하는 사용자 답장 기반 관계를 선호하는 사용자
범례 — 각 시안 페이지의 iOS 콜아웃은 사용할 SwiftUI 컨트롤, GAP 표시는 현재 RN repo에 아직 래퍼가 없어 신규 Swift 브리지나 추가 라이브러리가 필요한 항목을 의미합니다. 사용자가 시안을 선택하면 후속 작업으로 Supabase feedback_posts 테이블 마이그레이션, FeedbackBoardScreen RN 구현, 필요한 네이티브 브리지 작성을 진행할 수 있습니다.
