시안 B의 안티패턴 해결 방식

✅ 정보 과부화 해결

완료/보류 섹션은 기본 접힘(DisclosureGroup). 활성 상태만 펼쳐짐. 상태별 분류 자체가 시각 필터 역할.

✅ 불명확한 시작점 해결

Segmented control이 화면 최상단 + 풀폭 → "먼저 이걸 조작하세요"라는 암묵 지시. 새 제보는 우상단 `+` (Mail/Notes 관습).

✅ 플로우 단절 해결

상태 변화가 애니메이션으로 리스트 내 이동으로 표현(matchedGeometryEffect). 사용자는 진척을 시각적으로 체험 → 방문 유인 강화.

✅ 시각적 계층

검토중/진행중(full card) > 완료(요약 한 줄) > 보류(접힘) > 비공개(50% opacity). 중요도 = 시각 크기로 직접 매핑.

🔒 프라이버시 모델

각 상태 섹션 말미에 "비공개 · N건" 집계 행으로 표시(50% opacity). 커뮤니티 활성도는 보여주되 내용은 완전 차단. Admin 계정은 tap 시 상세 목록으로 push.

핵심 멘탈 모델

"내 제보들의 진행 상태 칸반". 사용자는 진척감(progress)에 끌려 재방문한다.
