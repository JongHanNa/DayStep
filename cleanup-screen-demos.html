시안 A: 오늘의 청소

에너지 레벨 적응형 마이크로태스크 — "한 번에 하나, 5분만"

🧠 행동설계 핵심

  • 에너지 적응형: 컨디션에 따라 태스크 수와 난이도 자동 조절 (힘듦=1-2개 초간단, 괜찮음=3-5개)
  • 한 번에 하나: 포커스 카드에 현재 태스크만 크게 표시 → 선택 마비 제거
  • 5분 규칙: 착수 저항 우회. 타이머와 연동해 "일단 시작" 강조
  • 스트릭 보상: 연속일 표시로 도파민 자극, 끊어져도 비판단적 리셋
  • 완료 큐: 나머지 태스크는 작게 — 전체 목록에 압도되지 않음

📱 네이티브 컨트롤

  • 에너지 선택기: UISegmentedControl 또는 iOS 26+ LiquidGlassMenu NATIVE
  • 타이머: 기존 TimerRing 컴포넌트 재사용 (ExecutionScreen 패턴) REUSE
  • 체크박스 애니메이션: withSequence(0.8 → 1.2 → 1) 스프링 (TodoCard 패턴) REUSE
  • 포커스 카드 전환: entering={ZoomIn} + exiting={FadeOut} Reanimated REANIMATED
  • 큐 리스트: SectionList with Layout.springify() REUSE

🏗 정보구조

  • 헤더: 화면 제목 + 설명
  • 에너지 선택기 (3단계)
  • 스트릭 바 (연속 기록)
  • 포커스 카드 (현재 1개 태스크 + 5분 타이머)
  • 대기열 (다음 태스크 2-3개, 흐리게)
  • 오늘 완료 섹션 (접히는 리스트)

🔄 태스크 플로우

  • 화면 진입 → 에너지 선택 → 오늘 추천 태스크 자동 생성
  • 포커스 카드 "시작하기" → 5분 타이머 실행 (ExecutionScreen과 동일 UX)
  • 완료 체크 → 축하 햅틱 + 다음 태스크 자동 승격
  • 스킵 가능 → 큐 다음으로 이동 (비판단적)
  • 모든 태스크 완료 → 축하 화면 + 스트릭 업데이트

시안 B: 구역별 순환 청소

요일별 구역 배정 — "월=침실, 화=거실, 수=욕실..." 예측 가능한 루틴

🧠 행동설계 핵심

  • 예측 가능성: 요일-구역 고정 매핑 → "오늘 뭐 청소하지?" 결정 제거
  • 시각적 완료감: 방별 청소 → 눈에 보이는 변화가 도파민 보상과 직결
  • 부분 완료 허용: 28% 완료도 기록됨 → 올오어낫싱 사고 방지
  • 일요일 자유: 쉬거나 밀린 구역 보충 → 유연성 확보
  • 월간 서페이싱: 딥클리닝은 월 1개씩 자동 배너로 상기
  • 3단계 난이도: 빠른 청소/꼼꼼 청소/정리비우기 → 에너지에 맞게 선택

📱 네이티브 컨트롤

  • 주간 캘린더: 기존 NativeWeekStripCalendar 재사용 REUSE
  • 난이도 탭: iOS 26+ LiquidGlass segmented / ScrollView 폴백 NATIVE
  • 체크리스트: SectionList with Layout.springify() REUSE
  • 구역 설정 시트: @gorhom/bottom-sheet (구역-요일 매핑 편집) BOTTOM SHEET
  • 월간 딥클리닝 상세: 네비게이션 push (스택) NAVIGATION
  • 주간 현황 차트: 네이티브 프로그레스 → NativeProgressCard 패턴 REUSE

🏗 정보구조

  • 헤더: 화면 제목 + 설명
  • 주간 캘린더 스트립 (완료 상태 도트)
  • 오늘의 구역 히어로 카드 (이름 + 진행률)
  • 난이도 탭 (빠른/꼼꼼/정리)
  • 체크리스트 (해당 난이도 항목들)
  • 월간 딥클리닝 배너 (필요시)
  • 이번 주 구역 현황 요약

🔄 태스크 플로우

  • 화면 진입 → 오늘 요일의 구역 자동 표시
  • 난이도 탭 선택 → 해당 구역의 항목 필터링
  • 체크 완료 → 구역 진행률 실시간 업데이트
  • 다른 요일 클릭 → 해당 구역으로 전환 (밀린 것 처리 가능)
  • 구역 100% → 축하 애니메이션 + 주간 현황 업데이트
  • FAB → 커스텀 항목 추가 (bottom sheet)

시안 C: 청소 허브

공간 청소 + 디지털 정리 + 물건 관리 — PDF 템플릿 3페이지를 하나로 통합

🧠 행동설계 핵심

  • 5분 랜덤 시작: 최상단 CTA → "뭐부터 할지" 결정 제거, 앱이 골라줌
  • 밀린 항목 서페이싱: 붉은 배너로 놓친 것 부드럽게 상기 (비판단적 톤)
  • 3탭 통합: PDF 템플릿의 공간 청소/디지털 정리/물건 관리를 탭으로 분리
  • 주기 뱃지: 매일/주간/월간/계절 주기 시각화 → 우선순위 직관적 파악
  • 확장/축소 카테고리: 한 방만 펼쳐서 집중 → 전체 목록 압도감 방지
  • 부분 완료 추적: 상단 요약에서 전체 진행률 확인 (73%도 좋은 거예요)

📱 네이티브 컨트롤

  • 3탭 세그먼트: LiquidGlassMenu (iOS 26+) / UISegmentedControl 폴백 NATIVE
  • 카테고리 확장/축소: SwiftUI DisclosureGroup 기반 네이티브 아코디언 NATIVE
  • FAB → 항목 추가: @gorhom/bottom-sheet (방 선택 + 항목 입력 + 주기 선택) BOTTOM SHEET
  • 밀린 항목 목록: 네비게이션 push 또는 bottom-sheet NAVIGATION
  • 5분 랜덤 시작: AnimatedPressable + 햅틱 + ExecutionScreen 타이머 연동 REUSE
  • 체크박스: withSequence 스프링 바운스 (TodoCard 패턴) REUSE
  • 긴 누르기 메뉴: LiquidGlassMenu — 항목별 수정/삭제/주기변경 NATIVE

🏗 정보구조

  • 헤더: 화면 제목 + 설명
  • "5분 랜덤 청소" CTA 버튼
  • 요약 통계 (주간 진행률 / 완료 수 / 밀린 수)
  • 밀린 항목 배너 (있을 때만)
  • 3탭: 공간 청소 | 디지털 정리 | 물건 관리
  • [공간 청소 탭] 방별 아코디언 → 체크리스트 + 주기 뱃지
  • [디지털 정리 탭] 기기별 아코디언 (이메일/PC/태블릿/스마트폰)
  • [물건 관리 탭] 방별 아코디언 (분기별 → 월간 분산)
  • FAB: 커스텀 항목 추가

🔄 태스크 플로우

  • 화면 진입 → 오늘 해야 할 주기 항목 자동 표시 (매일 항목 + 해당 요일 주간 항목)
  • "5분 랜덤" → 미완료 항목 중 랜덤 선택 → 타이머 시작
  • 방 카테고리 탭 → 확장 → 체크리스트 수행
  • 탭 전환 → 디지털 정리 / 물건 관리 체크리스트
  • FAB → 방 선택 + 항목 입력 + 주기 선택 (bottom sheet)
  • 밀린 항목 배너 → 밀린 목록 화면 → 처리/스킵/리스케줄

청소 주기 분석 — 일반 vs ADHD 전문 관점

1. 일반적인 청소 주기

주기항목소요 시간
매일
(5-15분)		 침대 정리, 설거지, 조리대 닦기, 간단 정리정돈, 쓰레기 처리 항목당 1-3분
주간
(각 15-30분)		 바닥 청소(청소기+물걸레), 화장실 전체, 먼지 제거, 침구 교체, 주방 깊은 청소 구역당 15-30분
월간
(각 20-40분)		 가전 내부, 창문, 수납 정리, 조명/천장, 배수구, 냉장고 내부 항목당 20-40분
계절/분기별 대청소, 커튼 세탁, 카펫 딥클리닝, 에어컨 필터, 계절 옷 교체, 창고 정리 반나절~하루

2. 분기별 관리가 ADHD에 부적합한 이유

PDF 템플릿의 "분기별 체크" 구조는 ADHD에 치명적입니다:

1. 시간 맹(Time Blindness): "3개월 후"를 실감 못 함 → 영원히 미뤄짐
2. 누적 = 압도감: 3개월 동안 쌓인 양을 보면 시작 자체를 포기
3. 실행 기능 과부하: "어디서부터?" 결정 자체가 에너지 소모
4. 도파민 부족: 보상 주기가 너무 김 → 동기부여 불가

3. ADHD 전문가 권장 주기

ADHD 최적 주기 = "매일 5분 + 주간 구역 순환 + 월간 1개 딥클리닝"

분기별 항목은 월간으로 쪼개서 분산 배치하는 것이 핵심입니다.

4. ADHD 행동설계 원칙 (앱 적용)

ADHD 어려움행동설계 대응앱 구현
작업 착수 장애 5분 규칙 + "일단 시작" 프레이밍 5분 타이머 CTA, 랜덤 시작 버튼
선택 마비 앱이 다음 할 일 선택 포커스 카드 (한 번에 1개만)
압도감 태스크 청킹 + 부분 표시 아코디언, 한 구역만 펼침
시간 맹 시각적 타이머 + 소요 시간 표시 TimeTimer 방식 원형 타이머
보상 부족 즉각적 피드백 + 스트릭 체크 시 햅틱 + 스트릭 바
완벽주의 → 회피 "Done > Perfect" 톤 부분 완료도 기록, 비판단적 UI
루틴 무너짐 리셋 용이한 체크리스트 밀린 항목 "스킵" 옵션, 재시작 쉬움

5. 방별 vs 작업유형별 — 어떤 게 좋은가?

ADHD에는 "방별"이 더 적합합니다

이유: 방 하나를 청소하면 눈에 보이는 변화가 즉각적 → 도파민 보상으로 직결됩니다. "먼지 닦기"를 집 전체에서 하면 어디서 끝인지 불분명하고, 성취감이 분산됩니다.

최적 하이브리드: 기본 구조는 방별, 각 방 안에서 작업유형별(빠른/꼼꼼/정리) 세분화
